## Bugs Found & Fixed

### 1. `conformVideo` ‚Äî missing `-movflags +faststart` and audio stream dropped
- **File:** `src/services/media-upload.service.ts`
- **Change:** Added `.map('0:a?')` (optional audio mapping) and `.audioCodec('aac')` so audio is preserved. Added `-movflags +faststart` to `outputOptions` array so the moov atom is placed at the start of the file (required for streaming and Meta mobile app compatibility per blueprint). Also upgraded `boxblur=20` to `boxblur=luma_radius=20:luma_power=4` for higher quality dreamy blur effect.
- **Revert:** Remove `.map('0:a?')`, `.audioCodec('aac')`, and `-movflags +faststart` from `outputOptions`. Change `boxblur=luma_radius=20:luma_power=4` back to `boxblur=20`.

### 2. Instagram carousel video items using wrong `media_type`
- **File:** `src/adapters/instagram.adapter.ts`
- **Change:** Reverted carousel item `media_type` from `'REELS'` back to `'VIDEO'`. The previous session's fix (v18‚Üív21) incorrectly changed the carousel item type to `'REELS'`. Per the Instagram Graph API: carousel video items must use `media_type: 'VIDEO'`, while only standalone video posts use `media_type: 'REELS'`.
- **Revert:** Change `media_type: isVideo ? 'VIDEO' : 'IMAGE'` back to `media_type: isVideo ? 'REELS' : 'IMAGE'` in the carousel loop.

### 3. Instagram smart-polling ‚Äî no timeout guard before `media_publish`
- **File:** `src/adapters/instagram.adapter.ts`
- **Change:** Added `if (!isReady) { throw new Error('...timed out...') }` check immediately after the polling `while` loop, before the `media_publish` API call. Previously, if polling expired after 3 minutes without `FINISHED`, the code fell through and called `media_publish` on a not-ready container, causing a silent failure or Meta API error.
- **Revert:** Remove the 5-line `if (!isReady)` block before the `media_publish` call.

### 4. Platform option key mismatch ‚Äî `instagramPostType` vs `postType`
- **Files:** `src/services/media-validator.service.ts`, `src/services/posting-engine.service.ts`
- **Change:** Backend was reading `options?.instagramPostType` and `platformOpts.instagramPostType`, but the frontend sends the option under the key `postType` (e.g. `platformOptions.INSTAGRAM.postType`). Changed both reads to `options?.postType` / `platformOpts.postType`. This bug caused Instagram REEL/STORY validation and Auto-Fix target ratios to always fall back to FEED settings.
- **Revert:** Change `postType` back to `instagramPostType` in both files.

### 5. Facebook Reel Auto-Fix ‚Äî wrong target aspect ratio
- **File:** `src/services/posting-engine.service.ts`
- **Change:** Added `else if (platform === 'FACEBOOK') { const type = platformOpts.postType || 'FEED'; if (type === 'REEL') targetRatio = 0.562; }` after the Instagram ratio block. Previously, Facebook Reels Auto-Fix defaulted to a square `1.0` ratio instead of the required 9:16 (`0.562`), producing non-conforming output.
- **Revert:** Remove the `else if (platform === 'FACEBOOK')` block (3 lines).

### 6. `/uploads` MIME type ‚Äî no explicit `Content-Type: video/mp4` enforcement
- **File:** `src/app.ts`
- **Change:** Replaced plain `express.static(...)` with a version that includes `setHeaders` callback. All `.mp4`, `.mov`, and `.webm` files are now explicitly served with `Content-Type: video/mp4`. This is required for Meta's bots to recognise and download the video without rejecting it.
- **Revert:** Replace the multi-line `express.static(..., { setHeaders: ... })` call with the original single-line `express.static(path.join(...))`.
