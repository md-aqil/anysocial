# Debug Changelog — Session 20260416_164910_435955

## Root Cause

**A phase-ordering deadlock in `FacebookAdapter.publish()` for Reels.**

Backend log evidence (`.backend.log`):
```
[FB PUBLISH] Attempt 1: Status is upload_complete
[FB PUBLISH] Attempt 2: Status is upload_complete
... (90 attempts, all 'upload_complete')
error: "Facebook publish failed: Facebook Reel processing timed out."
```

**Why it deadlocked:**
- `upload_complete` = Facebook has received the binary. This is expected after Phase 2.
- `ready` = Facebook has finished processing. This **only appears after Phase 3 (finish) is called**.
- The original code polled for `ready` for 3 minutes **before** calling Phase 3.
- Phase 3 was only called after the polling loop, which would never exit cleanly.
- Result: Phase 3 was never reached → video stayed at `upload_complete` forever → timeout → `PARTIALLY_FAILED`.

**Secondary issue:** The binary upload `axios.post` had no `maxBodyLength` or `maxContentLength`, which causes axios to silently cap or reject large video file bodies. Also had an undocumented `X-Entity-Type` header that is not part of Meta's rupload protocol.

## Changes Made

### 1. Restructured Phase ordering in Facebook Reel upload (Critical fix)
- **File:** `src/adapters/facebook.adapter.ts`
- **Change:** Replaced the incorrect 3-minute `ready`-polling loop (before Phase 3) with a short 30-second `upload_complete` confirmation loop, then immediately calls Phase 3 (finish). The original flow was:
  ```
  Phase 2 (binary upload) → Poll for 'ready' (3 min, never succeeds) → Phase 3 (never reached)
  ```
  The fixed flow is:
  ```
  Phase 2 (binary upload) → Confirm 'upload_complete' (≤30s) → Phase 3 (finish/publish)
  ```
- **Revert:** Restore the original `isFbReady` while-loop (90 attempts polling for `ready`) before the Phase 3 axios.post call. Remove the `uploadConfirmed` loop.

### 2. Added `maxBodyLength: Infinity` and `timeout` to binary upload
- **File:** `src/adapters/facebook.adapter.ts`
- **Change:** Added `maxBodyLength: Infinity`, `maxContentLength: Infinity` (prevents axios from silently capping large video bodies), and `timeout: 120000` (2-minute timeout for large file binary transfers) to the Phase 2 `axios.post` call.
- **Revert:** Remove `maxBodyLength`, `maxContentLength`, and `timeout` from the Phase 2 axios options.

### 3. Removed undocumented `X-Entity-Type: video/mp4` header
- **File:** `src/adapters/facebook.adapter.ts`
- **Change:** Removed the `X-Entity-Type: video/mp4` header from the Phase 2 binary upload request. This header is not part of Meta's documented rupload protocol and could cause unexpected behavior.
- **Revert:** Add `'X-Entity-Type': 'video/mp4'` back to the Phase 2 axios headers.

### 4. Added `timeout: 30000` to Phase 3 (finish) call
- **File:** `src/adapters/facebook.adapter.ts`
- **Change:** Added 30-second timeout to the Phase 3 finish call to prevent it from hanging indefinitely.
- **Revert:** Remove `{ timeout: 30000 }` from Phase 3 axios options.
